# Design: Filename-based game dates + schedule coverage

Date: 2026-06-26
Status: Approved (pending spec review)

## Problem

1. The internal `Date` column in TrackMan CSV exports is sometimes wrong, so
   game dates shown in the dashboard (game navigator, "Game N of N", game logs,
   trend baselines) can be off. The CSV **filenames** carry a reliable date
   prefix (`20260605-SanBrunoPark-1_unverified.csv`).
2. There is no way to see which scheduled games are missing TrackMan data. A
   coach (and a player) should be able to tell when a game has no CSV.

## Goals

- Use the filename date as the single source of truth for every game date in
  the app.
- Fetch the official 2026 schedule and show which scheduled games have a CSV and
  which do not — in a coach "Schedule" tab and as a lighter note in player
  accounts.
- Surface "orphan" CSVs (a CSV whose date matches no scheduled game).
- Degrade gracefully: a schedule-fetch failure must never break the app.

## Non-goals

- No per-player participation analysis (the player-side "missing CSV" list is
  team-level: scheduled games with no CSV at all, not "games this player sat").
- No editing of the schedule from the UI.
- No change to how metrics are computed — only the source of the game date.

## Design

### 1. Filename dates at ingestion (`sync_drive.R`)

`build_combined_csv()` reads each file in `data/game_csvs/` and binds them. For
each file, before binding, overwrite `Date` for all of its rows with the date
parsed from the filename's leading `YYYYMMDD`:

- Parse with a strict regex `^(\d{8})-`; on match, `Date <- as.Date(prefix, "%Y%m%d")`.
- If a filename has no valid date prefix, keep that file's internal `Date` and
  `message()` a warning naming the file (so it's visible in logs, not silent).

This is the only place the game date is set, so the combined `all_fall_25.csv`
(and everything downstream) uses filename dates with no other code changes.
`clean_trackman_data()` and the rest of the app are untouched.

The bundled `all_fall_25.csv` fallback is regenerated by this same function on
the next Drive sync, so production converges to filename dates on restart.

### 2. Schedule fetch + parse (new `schedule.R`)

```
fetch_schedule(url = SCHEDULE_URL) -> tibble | NULL
  columns: date (Date), opponent (chr), home_away ("Home"/"Away"),
           result (chr, e.g. "W, 5-4" or NA), played (lgl)
```

- Scrape with `rvest` (the schedule is in the raw HTML — confirmed, not
  JS-rendered).
- `played <- !is.na(result) & !grepl("not yet played", result, ignore.case=TRUE)`.
- Whole body wrapped in `tryCatch`; on any error return `NULL` and `message()`
  the reason. Callers treat `NULL` as "schedule unavailable".
- `SCHEDULE_URL` is a constant (this season's URL).

Called once at startup (like Drive sync) and cached in a reactive; refreshed by
the existing coach "Sync Data from Drive" button.

### 3. Coverage matching (`schedule.R` helper)

```
build_coverage(schedule, csv_files) -> list(games=tibble, orphans=chr)
```

- `csv_dates` = dates parsed from `data/game_csvs/` filenames (same strict regex);
  keep a count of CSVs and total pitches per date.
- For each schedule row, `data_status`:
  - `played` & date in csv_dates  -> "tracked" (with pitch count)
  - `played` & not in csv_dates   -> "No CSV"
  - not `played`                  -> "upcoming"
- `orphans` = csv_dates that match no scheduled game's date (formatted as
  `date — filename`).

### 4. Coach "Schedule" tab

New `tabPanel("Schedule", ...)` in the coach view:

- Summary line: "Tracked X of Y played games."
- Table: Date | Opponent | H/A | Result | Data (✓ N pitches / **No CSV** /
  upcoming). "No CSV" rows visually flagged (red).
- If `orphans` non-empty: an "Unmatched CSV files" note listing them.
- If schedule is `NULL`: "Schedule unavailable — could not reach
  collegeseagulls.com." (rest of app unaffected.)

### 5. Player account "missing data" note

In the player portal, a compact, collapsible note (not a full table) listing
scheduled **played** games that have **No CSV**, e.g.:

> Missing game data (no CSV yet): Jun 3 vs Menlo Park, Jun 7 @ Walnut Creek, …

Uses the same `build_coverage()` result. Team-level (not player-specific). If the
schedule is unavailable, the note is omitted entirely.

## Error handling

- Drive sync: already wrapped (unchanged).
- Schedule fetch/parse failure -> `NULL` -> "unavailable" messaging; no crash.
- Filename without a date prefix -> internal date kept + logged warning.

## Testing

Before wiring UI:
1. Filename-date parser: unit-check against the real `data/game_csvs/`
   filenames (all 15 currently present parse to the expected dates; a name with
   no prefix falls back).
2. `fetch_schedule()`: returns the expected ~37 rows with correct
   played/upcoming split against the live page.
3. `build_coverage()`: against the real schedule + CSV files, assert the known
   missing games (Jun 3, 7, 9, 16, 17 — all away, no CSV) are "No CSV" and the
   orphans (the extra `20260622` and second `20260624` files) are flagged.

Then verify the coach tab and the player note render on the running app.

## Dependencies

- `rvest` (HTML scraping) — confirmed installed locally and reads the live page.
  It is a new runtime dependency; `rsconnect` auto-detects it from `schedule.R`,
  so the next deploy bundles it. No change needed to `deploy_app.R`.

## Files touched

- `sync_drive.R` — filename-date override in `build_combined_csv()`.
- `schedule.R` — new: `fetch_schedule()`, `build_coverage()`, `SCHEDULE_URL`.
- `global.R` — `source("schedule.R")`.
- `server.R` — schedule/coverage reactives; coach Schedule tab; player note.
